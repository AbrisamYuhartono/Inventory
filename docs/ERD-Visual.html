Key Relationships

Users → Lending_Records

One user can have many lending records (1:N)

Items → Lending_Records

One item can have many lending records over time (1:N)

Rooms → Items

One room can contain many items (1:N)

Categories → Items

One category can have many items (1:N)

Lending_Records → Lending_Documents

One lending record can have multiple documents (1:N)

Users → Notifications

One user can receive many notifications (1:N)
